# Licensing Protection Framework Design

**Date:** 2026-04-07
**Status:** Approved
**Approach:** Approach A (Grafana Model + AI-era protections)

## Context

Frontman is entering enterprise whitelabel deals with hosting providers. The current licensing (AGPL-3.0 on server, Apache-2.0 on client libs) lacks protections for:

- Partners forking and competing without contractual obligations
- AI coding assistants laundering AGPL code into non-AGPL projects
- Partners using AI to analyze architecture and rebuild competitors
- Brand confusion from forks using the Frontman name
- Inability to dual-license due to missing CLA

All server code is currently 100% founder-written. No external contributors yet.

## Decisions

- Stay AGPL-3.0 on server, Apache-2.0 on client libs
- CLA bot (CLA Assistant) for contributor agreements
- Full framework: CLA, trademark policy, AI supplementary terms, commercial license template
- CLA grants relicensing rights (enables dual licensing and future license changes)

## Components

### 1. Contributor License Agreement (CLA.md)

Individual + Corporate CLA in a single document.

**Key clauses:**
- Copyright grant: perpetual, irrevocable, worldwide, non-exclusive, royalty-free license to reproduce, modify, sublicense, and distribute under any terms
- Patent grant: covers patents necessarily infringed by contributions
- Relicensing right: explicit permission to offer contributions under alternative licenses
- Original work warranty: contributor authored it or has right to submit
- No obligation: Frontman AI not obligated to use any contribution

**Mechanics:**
- CLA Assistant GitHub bot checks PRs
- First-time contributors sign via GitHub OAuth
- Signatures stored in repo
- CI check blocks merge until signed

### 2. Trademark Policy (TRADEMARK.md)

**Protected marks:** "Frontman" wordmark, "Frontman AI" company name, Frontman logo, frontman.ai domain.

**Permitted uses:**
- "Built with Frontman" / "Powered by Frontman" attribution
- Referencing Frontman in blog posts, articles, talks
- Linking to official repo/site
- Fork for personal/internal use with unofficial disclaimer

**Prohibited uses:**
- Naming a commercial product "Frontman" or "Frontman [Something]"
- Using logo to imply endorsement/affiliation
- Using "Frontman" in a domain name for competing service
- Modifying the logo
- Using Frontman branding in AI training datasets
- AI-generated content implying Frontman endorsement
- Fine-tuning models on Frontman materials and presenting outputs as Frontman
- AI chatbots/agents representing themselves as Frontman without a license

**Fork naming:** Forks must use a distinct name. "Based on Frontman" is okay; "Frontman Plus" is not.

**Whitelabel exception:** Commercial license holders may use agreed-upon branding per their agreement.

### 3. AI Supplementary Terms (AI-SUPPLEMENTARY-TERMS.md)

Added under AGPL Section 7 as supplementary terms.

**Clause 1 — AI Training Restriction:**
Source code may not be used as training, fine-tuning, or evaluation data for ML/AI systems without prior written consent.

**Clause 2 — AI-Assisted Reproduction Restriction:**
Using AI tools to analyze, reverse-engineer, or reproduce the architecture, algorithms, data structures, or design patterns for the purpose of creating a substantially similar or competing product is a violation.

**Clause 3 — Output Restriction:**
Code generated by AI systems that is substantially derived from this project remains subject to AGPL-3.0, regardless of derivation method.

**Legal basis:** AGPL Section 7(b) and 7(c) — preservation of legal notices and prohibition of misrepresentation of origin.

**Limitations:** Cannot stop Copilot/Cursor indexing, cannot prevent human learning, cannot override fair use, cannot enforce against unknown actors. These terms serve as deterrent and litigation foundation.

### 4. Commercial License Template (COMMERCIAL-LICENSE.md)

Template/summary in repo. Actual executed agreements are separate legal documents.

**Grant of rights:**
- Use, modify, deploy server for partner's own product/service
- Remove AGPL obligations (no source disclosure)
- Whitelabel/rebrand per trademark agreement
- Redistribute to end customers as part of hosted service only (not standalone)

**Restrictions:**
- Non-compete: cannot offer standalone competing AI agent platform
- No sub-licensing of source code
- No redistribution of source to partner's customers
- AI cloning prohibition: cannot use AI to analyze code for building similar product
- Survival: AI cloning and non-compete survive termination for 24 months

**Partner obligations:**
- Attribution: "Powered by Frontman" unless waived in agreement
- Security notification: must report vulnerabilities discovered
- Audit rights: annual compliance verification
- Upstream contributions: encouraged (not mandated) for non-partner-specific improvements

**Commercials (blanks):** License fee, territory, term/renewal, support SLA, termination terms.

**Termination consequences:**
- Stop using code within 90 days
- Transition period for end customers (negotiated)
- Source code deleted or returned
- Non-compete survives 24 months
- AI cloning prohibition survives 24 months
- Confidentiality survives indefinitely

### 5. License Hygiene

| Item | Change |
|------|--------|
| Server AGPL version | Pin to AGPL-3.0-only (drop "or later") |
| Server mix.exs | Add `licenses: ["AGPL-3.0-only"]` |
| Marketing LICENSE | Replace MIT/Oxygenna with Apache-2.0/Frontman AI |
| CONTRIBUTING.md | Require CLA, reference CLA.md, explain dual-license |
| Server source file headers | Add AGPL + AI terms reference to .ex/.exs files |
| CLA bot workflow | Add GitHub Actions workflow for CLA Assistant |

**Source file header for server:**
```elixir
# Frontman Server
# Copyright (C) 2025 Frontman AI
#
# Licensed under the AGPL-3.0 — see LICENSE for details.
# Additional terms apply — see AI-SUPPLEMENTARY-TERMS.md
```

## Important Caveats

- All documents should be reviewed by an IP/open-source attorney before enforcement
- AI supplementary terms under AGPL Section 7 are legally untested
- Commercial license template is a starting point, not a final legal instrument
- Trademark registration should be filed formally (the policy assumes marks are claimed, not necessarily registered)
